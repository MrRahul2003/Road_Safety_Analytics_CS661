# Road Accident Severity — Visual Analytics System

A web-based visual analytics application for the **RTA road-traffic-accident dataset**
(Addis Ababa; ~12k records, 32 attributes, target = `Accident_severity` ∈ {Slight
Injury, Serious Injury, Fatal injury}). Built for CS661 — Big Data Visual Analytics.

The system is organised around four design pillars:

| Pillar          | Where it lives                                                            |
|-----------------|---------------------------------------------------------------------------|
| **Geo plotting**| *Geospatial* view — crash-density bubble map + risk ranking by area       |
| **Insights**    | *Insights* + *Causal* views — who/where/under-what-conditions, cross-filtered |
| **Colorblind**  | Global — Okabe-Ito / Tol-Bright CVD-safe palettes (switchable) + cividis scales |
| **Predictive**  | *Predictor* view — in-browser multinomial logistic-regression severity forecaster |

This is a **web application**, not a scrolling dashboard: a left-nav switches between
eight analytical views, and **every chart is a cross-filter** — click a bar / cell / bubble /
legend and the selection propagates to all other views (OR within a field, AND across
fields). A live `X / 12,000 records` counter and a removable filter-chip bar sit in the
top bar.

## Architecture

```
road-safety-analytics/
├── data_cleaning/
│   ├── clean_and_prepare.py     # single-file pipeline: clean → derive → analyse → model → export
│   └── requirements.txt
├── data/
│   ├── raw/                     # ← put the real "RTA Dataset.csv" here
│   └── cleaned/                 # ← rta_cleaned.csv is written here (separate folder)
└── frontend/                    # Vite + React app
    ├── public/data/             # records.json, meta.json, model.json (generated by the pipeline)
    └── src/
        ├── data/store.jsx       # context: loads feeds, owns filter state, memoised cross-filter
        ├── theme.js             # CVD-safe palettes + cividis
        └── components/          # Overview, Geospatial, Temporal, Insights, Causal, Predictor
```

The Python pipeline runs **offline once**; it writes a cleaned CSV to `data/cleaned/`
**and** three JSON feeds into `frontend/public/data/`. The React app loads those feeds
and does all cross-filtering client-side (12k rows is tiny). The severity model is trained
in Python; its weights are exported to `model.json` and inference (softmax) runs live in
the browser as you change the Predictor inputs.

## How to run

### 1 · Clean the data + train the model (Python ≥ 3.9)

```bash
cd data_cleaning
pip install -r requirements.txt
python clean_and_prepare.py
```

- **Place the real `RTA Dataset.csv` in `data/raw/` first.** If no raw CSV is found, the
  script generates a clearly-flagged **synthetic placeholder** so the whole app still runs
  end-to-end — but those numbers are meaningless. Drop in the real file and re-run for real
  insights and real model metrics.
- Outputs: `data/cleaned/rta_cleaned.csv`, and `frontend/public/data/{records,meta,model}.json`.

### 2 · Run the frontend (Node ≥ 18)

```bash
cd frontend
npm install
npm run dev          # http://localhost:5173
# or: npm run build && npm run preview
```

## Data cleaning techniques (all in `clean_and_prepare.py`)

1. **T1** strip/normalise column names
2. **T2** strip + collapse internal whitespace in all string cells
3. **T3** standardise null tokens (`na`, `unknown`, `''`, …) → `NaN` (keeps meaningful `Other`)
4. **T4** drop exact duplicate rows
5. **T5** coerce count columns (vehicles / casualties) to numeric
6. **T6** cap numeric outliers at the 99th percentile
7. **T7** impute missing categoricals → `Unknown`
8. **T8** canonicalise the target labels, drop unmappable target rows
9. **T9** consolidate rare categories (<0.5%) → `Other` (skips the target **and** the
   high-cardinality `Time` column so hour-of-day survives)
10. **Derived features:** `Hour`, `Time_of_day`, `Is_weekend`, `Severity_rank`
11. **Aggregations:** severity-rate tables by area / cause / weather / light / age /
    experience, each with a weighted risk score (serious + 3×fatal).

## The analytical tasks / data story (≥ 5)

1. **Temporal risk windows** — when do accidents happen and when do they turn severe?
   (*Temporal* — day×hour cividis heatmap with volume/risk toggle + hourly severity bars.)
2. **Driver demographics** — which age bands and experience levels carry the most lethal
   outcomes? (*Insights* — severity-composition small multiples, sorted by risk.)
3. **Environmental conditions** — effect of weather, light and road surface on severity.
   (*Insights*.)
4. **Geospatial concentration** — which area types are accident hot-spots and which are
   most lethal per crash? (*Geospatial* — bubble map sized by volume, coloured by risk +
   ranked table.)
5. **Causal frequency-vs-lethality** — separating common-but-mild from *rare-but-deadly*
   causes/collision types so interventions target the right thing. (*Causal* — stacked
   bars + a frequency×fatal-rate scatter with a mean-fatality reference line.)
6. **Predictive severity** — given a scenario (time, road, weather, driver, cause…),
   forecast the probability of each severity class and show which inputs push the outcome.
   (*Predictor*.)

**Story:** raw accident counts are dominated by minor injuries, so counting crashes
misranks the real danger. By switching from *volume* to a *risk* lens (weighting serious
and fatal outcomes) and cross-filtering across time, place, environment, driver and cause,
the system surfaces the specific conditions under which an ordinary crash becomes a fatal
one — and the predictor turns that into a what-if tool.

## Model

Multinomial **logistic regression** (`class_weight='balanced'`, one-hot features rebuilt
identically in the browser). On the bundled **synthetic placeholder** data it reports
~0.61 accuracy / 0.51 macro-F1 — *placeholder numbers only*. Re-running the pipeline on
the real `RTA Dataset.csv` produces the real metrics, which the Predictor view reads
straight from `model.json` (accuracy, macro/weighted-F1, and per-class precision / recall /
F1 / support). Class-balanced weighting is deliberate: on this heavily Slight-Injury-skewed
dataset, recall on the rare Fatal/Serious classes matters more than raw accuracy.

## Notes & honest caveats

- **No GPS in the dataset.** Accident *location* is recorded only as area **categories**
  (`Area_accident_occured`), so the "geospatial" view is an area-based crash-density map,
  not a lat/long map. This is stated in the UI.
- **Colorblind-safe by construction:** all categorical encodings use Okabe-Ito or
  Tol-Bright palettes (toggle in the sidebar); all sequential encodings use **cividis**.
- `records.json` is ~7.7 MB — fine to load locally.
- Libraries: React + Recharts + d3-scale/d3-array/d3-scale-chromatic/d3-geo. No PowerBI /
  Tableau / Superset.

### Two extra views beyond the core six

- **Patterns** — chart types not used elsewhere: a Cramér's V *factor-association ranking*
  (which variables most relate to severity, computed live from the filtered data), a *treemap*
  of causes (area = volume, colour = fatal rate), a *radar* "risk fingerprint" of danger
  conditions vs the baseline, and a *casualty-count histogram*.
- **India** — a state choropleth (`frontend/public/data/india_states.geojson`, dissolved from
  district boundaries) shaded by accident rate, with click-to-drill that updates its KPIs and
  state ranking. **This is a separate dataset** — `frontend/public/data/india_accidents.json`,
  sourced from *MoRTH, Road Accidents in India 2022* — and is independent of the Addis Ababa
  records that drive every other view (the RTA dataset has no Indian geography). Top-state
  figures are verified against the MoRTH report; edit the JSON to refine smaller UTs.
